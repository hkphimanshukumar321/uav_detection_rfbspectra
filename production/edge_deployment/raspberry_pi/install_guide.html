# Installation Guide – Layman Friendly DroneRFB‑Spectra – Quick‑Start Guide
🚁

DroneRFB‑Spectra Quick‑Start

1What you need

  • Raspberry Pi 4 (4 GB) – $55
  • RTL‑SDR Blog V3 dongle – $25
  • 2.4 GHz antenna – $15
  • Micro‑SD card (32 GB) – $10
  • Power bank (5 V 3 A) – $20

2Connect the hardware

Plug the RTL‑SDR into one of the Pi’s USB ports, attach the antenna, and insert the micro‑SD card.

3Install the software

  1. Power up the Pi and open a terminal. 
    sudo apt update && sudo apt upgrade -y
  2. Clone the repository and install dependencies. 
    git clone https://github.com/yourorg/DRONE-RFB-SPECTRA.git\ncd DRONE-RFB-SPECTRA/production\npip install -r requirements_production.txt
  3. Build the Docker image (optional, but the easiest way). 
    docker compose up -d
  4. If you prefer a native install, run the provided installer script. 
    cd edge_deployment/raspberry_pi\nchmod +x install.sh\n./install.sh

4Run the detector

After installation the system is ready to start detecting drones.

python3 detect_realtime.py --freq 2.437e9 --gain 40

A tiny web dashboard will be available at http://YOUR_PI_IP:8080. Open it in any browser (desktop, phone, tablet).

5What you see on the dashboard

Header – system status, start/stop button

Live detection – drone type, confidence, timestamp

History – last 10 detections with latency

6FAQ & Troubleshooting

Why is the detection confidence low?

Low confidence usually means the RF environment is noisy or the antenna is not optimally positioned. Try moving the antenna away from metal objects and increase the SDR gain.

The web dashboard does not load.

Make sure the Docker container (or native Python process) is running. Check the container logs with docker logs dronerfb_spectra or run ps aux | grep dashboard.py.

Can I use a different SDR?

Yes – any SDR that provides raw IQ samples at ≥2 MS/s works. Update the detect_realtime.py script to use the appropriate driver.
