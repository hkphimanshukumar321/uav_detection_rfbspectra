🔄 How It Works

End-to-end pipeline from RF signals to drone detection

1

📡 RF Signal Capture

RTL-SDR or USRP device captures IQ samples at 2.4/5.8 GHz ISM bands (50ms windows at 20 MS/s)

Input: Complex IQ samples [120,000 samples]
Frequency: 2.437 GHz (ISM band)
Duration: 50ms per window
2

📊 Spectrogram Generation

Short-Time Fourier Transform (STFT) converts time-domain IQ data to frequency-time representation

Process: IQ → STFT (Hann window, 512 FFT)
Output: 512×512 spectrogram matrix
Time: ~10-15ms on Raspberry Pi
3

🎨 Binary Preprocessing

Novel pipeline: Grayscale conversion → Resize to 64×64 → Otsu's automatic binarization

Key Innovation: Binarization preserves RF signal characteristics (frequency hops, duty cycles) while enabling extreme compression for edge devices.
4

🧠 RF-DenseNet Inference

Lightweight DenseNet processes binary spectrogram through dense blocks with feature reuse

Model: RF-DenseNet (150K params, INT8 quantized)
Input: 64×64×1 binary tensor
Output: 25 class probabilities
Latency: <5ms on Pi, <2ms on Jetson

🔬 Research Methodology

Comprehensive experimental validation

Systematic Ablation Studies

  • ► Growth Rate: Tested [4, 8, 12] - optimal at 8 for edge constraints
  • ► Compression: [0.25, 0.5, 0.75] - 0.5 balances accuracy and efficiency
  • ► Network Depth: (2,2,2) to (4,4,4) - (3,3,3) optimal
  • ► Batch Size: [8, 16, 32, 64] - proves small-batch viability

Validation Experiments

  • ✓ 5-Fold Cross-Validation: 96.5% ± 0.3% accuracy (robust)
  • ✓ SNR Robustness: Tested at 7 noise levels (0-30 dB)
  • ✓ Binarization Ablation: Otsu outperforms mean/adaptive
  • ✓ Statistical Significance: McNemar's test (p < 0.001)
